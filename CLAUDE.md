# Claude Code — プロジェクトルール

## GUIDELINES.md の更新ルール

**以下のいずれかに該当する変更を行ったら、必ず `GUIDELINES.md` を更新すること。**

- レイアウト定数（座標・サイズ・フォントサイズ）の変更
- カラムレイアウトの列幅・配置の変更
- 描画レイヤー順の変更
- 新しい描画関数の追加
- データモデル（`models.py`）の変更
- テーマ定義の変更
- 既知の注意事項・設計判断が追加された場合
- テスト方法の変更

更新は「該当セクションを正確に書き換える」形で行う。履歴や追記ではなく、常に現状を正確に反映した状態を保つこと。

---

## 基本方針

- **正規化座標系を維持する**: レイアウト値はすべて `layout.py` の定数として定義し、各レンダラーが変換する。ハードコードした px 値を描画関数に直接書かない。
- **Pillow と ReportLab は並行開発**: Pillow（プレビュー）と ReportLab（PDF）の両方が同じ `layout.py` の定数を使う。片方だけ変更した場合、もう片方も対応が必要か確認すること。
- **テストは `python3 test_preview.py` で目視確認**: 変更後は必ずテストし、生成画像を確認してから報告すること。

---

## 実行方法

```bash
# テスト（プレビュー画像生成）
python3 test_preview.py

# アプリ起動
streamlit run app.py
```

---

## プロジェクト固有の注意事項

- 縦書きは「横書きで描画 → `-90°` 回転」で実装。`+90°` にしないこと（逆向きになる）。
- `PROG_TOP = 0.180` は年度テキスト用スペース確保のための意図的な値。変更時は GUIDELINES.md の「5. 縦方向レイアウト」と「PROG_TOP の根拠」も更新すること。
- Streamlit の `app.py` では関数定義を呼び出し箇所より前に配置すること（上から順に実行されるため）。
